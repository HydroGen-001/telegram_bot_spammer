# АВТОРИЗАЦИЯ С ПРОВЕРКОЙ ПРОКСИ

## 🚀 Быстрый старт

### Шаг 1: Настройте .env

```bash
# API Telegram
API_ID=12345
API_HASH=abc123def456
PHONE=+79990000000

# Прокси (обязательно!)
PROXY_ENABLED=true
PROXY_HOST=mobpool.proxy.market
PROXY_PORT=1337
PROXY_USERNAME=your_login
PROXY_PASSWORD=your_password
```

### Шаг 2: Запустите авторизацию с проверкой

```bash
python login_with_proxy.py
```

### Шаг 3: Выберите опцию

```
1. Войти с проверкой прокси     ← РЕКОМЕНДУЕТСЯ
2. Проверить прокси (без входа)
3. Проверить текущий IP
0. Выход
```

## ✅ ЧЕКЛИСТ

Перед запуском проверьте:

- [ ] `PROXY_ENABLED=true` в .env
- [ ] `PROXY_HOST` указан верно
- [ ] `PROXY_PORT` указан верно
- [ ] `PROXY_USERNAME` и `PROXY_PASSWORD` верные
- [ ] API credentials заполнены

После входа проверьте:

- [ ] Telegram прислал уведомление о входе
- [ ] В уведомлении указан IP прокси (не ваш реальный!)
- [ ] Сессия сохранена в `sessions/userbot.session`

## 🆘 ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК

### Telegram показывает ваш реальный IP

**Причина:** Сессия была создана без прокси

**Решение:**
1. Запустите `python login_with_proxy.py`
2. Выберите опцию 1
3. На вопрос "Выйти и войти заново?" ответьте `y`

### Прокси не подключается

**Причина:** Неверные данные прокси или прокси недоступен

**Решение:**
1. Проверьте .env (логин, пароль, порт)
2. Проверьте доступность прокси: `python login_with_proxy.py` → опция 2
3. Свяжитесь с поставщиком прокси

### FloodWait (ждите N секунд)

**Причина:** Слишком много попыток входа

**Решение:** Подождите указанное время и попробуйте снова
