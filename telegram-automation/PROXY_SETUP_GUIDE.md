# НАСТРОЙКА ПРОКСИ ДЛЯ TELEGRAM

## ⚠️ ПРОБЛЕМА
Если Telegram присылает уведомление о входе с **вашего реального IP**, а не с IP прокси — прокси не работает.

## ✅ РЕШЕНИЕ

### 1. Проверьте .env

```bash
# Прокси (обязательно включите!)
PROXY_ENABLED=true
PROXY_HOST=mobpool.proxy.market
PROXY_PORT=1337
PROXY_USERNAME=your_username
PROXY_PASSWORD=your_password
```

### 2. Авторизуйтесь ЧЕРЕЗ ПРОКСИ

**Вариант A: Через главное меню start.py**

```bash
python start.py
```

1. Выберите **"2. Войти через прокси"**
2. Введите телефон
3. Введите код из Telegram
4. Проверьте уведомление — должен быть IP прокси!

**Вариант B: Через тестовый скрипт**

```bash
python test_proxy_connection.py
```

### 3. Проверка прокси

После входа Telegram должен прислать уведомление:
```
Новый вход в Telegram
IP: xxx.xxx.xxx.xxx (прокси)
```

Если видите **свой реальный IP** — прокси не работает!

---

## 🔧 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Прокси не работает

**Проверьте формат proxy в коде:**

```python
# ПРАВИЛЬНО для SOCKS5:
proxy=('socks5', 'ip_адрес', порт, 'логин', 'пароль')

# ПРАВИЛЬНО для HTTP:
proxy=('http', 'ip_адрес', порт, 'логин', 'пароль')
```

**Проверьте доступность прокси:**

```bash
python test_proxy_connection.py
```

### Прокси работает, но Telegram показывает реальный IP

**Проблема:** Сессия была создана БЕЗ прокси

**Решение:**
1. Выйдите из сессии через меню
2. Авторизуйтесь заново через прокси (пункт 2)

### FloodWait при авторизации

**Проблема:** Слишком много попыток входа

**Решение:** Подождите время, указанное в ошибке

## 🔍 КАК ЭТО РАБОТАЕТ

1. **Создание клиента с прокси:**
   ```python
   client = TelegramClient(
       session,
       api_id,
       api_hash,
       proxy=('socks5', ip, port, username, password)
   )
   ```

2. **Подключение:**
   ```python
   await client.connect()  # ← Использует прокси!
   ```

3. **Авторизация:**
   ```python
   await client.send_code_request(phone)  # ← Через прокси!
   await client.sign_in(phone, code)      # ← Через прокси!
   ```

4. **Проверка:**
   - Telegram присылает уведомление
   - В уведомлении должен быть IP прокси
