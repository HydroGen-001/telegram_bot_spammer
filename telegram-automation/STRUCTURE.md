## 📌 Основные файлы

| Файл | Назначение | Когда использовать |
|------|-----------|-------------------|
| `start.py` | Главное меню | 🔥 **Основной файл** для запуска |
| `login_with_proxy.py` | Авторизация с прокси | Для входа через прокси (мобильный IP) |
| `multi_account_start.py` | Мультиаккаунтное меню | Для работы с несколькими аккаунтами |
| `src/main.py` | Автоматическая система | Для полноценной автоматизации |

## 📁 Папки

### 🔑 core/
Ядро системы с модульной архитектурой:
- `client.py` — обёртка над TelegramClient с поддержкой прокси
- `broadcaster.py` — логика рассылки сообщений
- `auth.py` — функции авторизации

### 👥 multi_account/
Мультиаккаунтная рассылка:
- `config.py` — загрузка и валидация конфигурации
- `manager.py` — управление несколькими клиентами
- `broadcaster.py` — параллельная отправка сообщений

### 🤖 src/
Автоматизированная система с полной архитектурой:
- `main.py` — точка входа, инициализация всех компонентов
- `core/` — ядро (клиент, конфиг, БД, логгер)
- `modules/` — модули (отправка, ответы, защита, планировщик)
- `utils/` — вспомогательные функции

### ⚙️ config/
Конфигурационные файлы:
- `templates.json` — шаблоны сообщений (текст + фото)
- `chats.json` — список чатов для рассылки

### 🔐 sessions/
Сессии Telegram:
- `userbot.session` — основная сессия
- `account_*.session` — сессии дополнительных аккаунтов

⚠️ **Не синхронизируется через Git** (копировать вручную)

### 📷 photos/
Фотографии для рассылки:
- `photo_*.jpg` — изображения для отправки

### 📊 logs/
Логи работы:
- `broadcast_log.json` — статистика рассылок
- `*.log` — текстовые логи

⚠️ **Не синхронизируется через Git**

### 🗄️ database/
SQLite база данных:
- `automation.db` — история сообщений, статистика

⚠️ **Не синхронизируется через Git**

### 🧪 tests/
Тестовые и проверочные скрипты:
- `test_*.py` — тесты функциональности
- `check_*.py` — проверки подключения, прокси и т.д.

### 📚 docs/
Документация проекта:
- `GIT_GUIDE.md` — работа с Git
- `QUICKSTART.md` — быстрый старт
- `ROADMAP.md` — план развития
- `TROUBLESHOOTING.md` — решение проблем
- и другие

## 🔒 Безопасность

### Файлы, которые НЕ должны попадать в Git:

| Файл/Папка | Причина |
|-----------|---------|
| `.env` | Содержит API ключи |
| `sessions/*.session` | Доступ к Telegram аккаунту |
| `database/*.db` | Личные данные, история |
| `logs/*.log`, `logs/*.json` | Логи работы |
| `__pycache__/` | Кэш Python |

Эти файлы уже указаны в `.gitignore`.
