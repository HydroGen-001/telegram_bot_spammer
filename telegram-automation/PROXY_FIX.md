# ИСПРАВЛЕНИЕ ПРОКСИ (Telethon 1.42+)

## ❌ ПРОБЛЕМА

Telegram показывает ваш реальный IP (Пермь), а не IP прокси.

## ✅ РЕШЕНИЕ

В Telethon 1.42+ изменился формат параметра `proxy`.

**Старый формат (НЕ РАБОТАЕТ):**
```python
proxy=('socks5', '1.2.3.4', 1080, 'user', 'pass')
```

**Новый формат (РАБОТАЕТ):**
```python
proxy={
    'proxy_type': 'socks5',
    'addr': '1.2.3.4',
    'port': 1080,
    'username': 'user',
    'password': 'pass'
}
```

## 📊 СРАВНЕНИЕ ФОРМАТОВ

| Параметр | Старый формат | Новый формат |
|----------|---------------|--------------|
| Тип | Кортеж `tuple` | Словарь `dict` |
| Синтаксис | `('socks5', ...)` | `{'proxy_type': 'socks5', ...}` |
| Telethon | < 1.40 | ≥ 1.42 |
| Работает | ❌ Нет | ✅ Да |

## 🆘 ЕСЛИ ВСЁ ЕЩЁ НЕ РАБОТАЕТ

### 1. Проверьте версию Telethon

```bash
pip show telethon
```

**Должно быть:** 1.42.0 или выше

### 2. Перезапустите Python

Иногда помогает перезапуск:
```bash
# Закройте все окна Python
# Запустите заново
python login_with_proxy.py
```

### 3. Очистите кэш

```bash
# Удалите __pycache__
rd /s /q __pycache__
```

### 4. Проверьте прокси отдельно

```bash
python test_proxy_real.py
```

Если подключается — прокси работает.

### 5. Свяжитесь с поставщиком прокси

Убедитесь что:
- Прокси активен
- Порт открыт (10000-10999)
- Логин/пароль верные
