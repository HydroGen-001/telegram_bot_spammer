# 📋 Инструкция: Распределение и вступление в чаты

## 🎯 Назначение

Скрипты для **массового распределения 3500 чатов** по аккаунтам и **постепенного вступления** с защитой от бана.

---

## 📁 Структура файлов

```
telegram-automation/
├── accounts.json                    # Конфигурация аккаунтов
├── config/
│   ├── chats.json                   # Текущая база чатов (источник)
│   ├── chats_base.json              # Общая база (можно создать)
│   └── accounts/                    # Папка с аккаунтами
│       ├── 1/
│       │   ├── chats.json           # Чаты для аккаунта 1
│       │   └── progress.json        # Прогресс вступления
│       ├── 2/
│       │   ├── chats.json           # Чаты для аккаунта 2
│       │   └── progress.json        # Прогресс вступления
│       └── ...
└── scripts/
    ├── distribute_chats.py          # Распределение чатов
    └── join_chats.py                # Вступление в чаты
```

---

## 🚀 Быстрый старт

### **Шаг 1: Подготовьте базу чатов**

У вас уже есть `config/chats.json` с чатами.

**Опционально:** Переименуйте в `config/chats_base.json` для общей базы.

---

### **Шаг 2: Распределите чаты по аккаунтам**

```bash
cd c:\Users\ACER\OneDrive\Desktop\project\telegram-automation
python scripts/distribute_chats.py
```

**Меню:**
```
1. Распределить чаты          → Создаст папки accounts/1/, accounts/2/, ...
2. Изменить кол-во чатов      → По умолчанию 200 чатов/аккаунт
3. Показать статистику        → Показать текущую базу
4. Открыть папку              → Открыть config/accounts/
0. Выход
```

**Что происходит:**
- Скрипт читает `chats.json` (3500 чатов)
- Читает `accounts.json` (список аккаунтов)
- Распределяет по **200 чатов на аккаунт**
- Создаёт файлы:
  - `config/accounts/1/chats.json`
  - `config/accounts/2/chats.json`
  - и т.д.

---

### **Шаг 3: Вступайте в чаты**

```bash
python scripts/join_chats.py
```

**Меню:**
```
1. Вступить в чаты            → Запуск вступления
2. Настройки лимитов          → Изменить дневной лимит
3. Показать прогресс          → Статистика по аккаунтам
4. Сбросить прогресс          → Начать заново
5. Открыть папку              → Открыть config/accounts/
0. Выход
```

**Процесс:**
1. Выберите аккаунт (например, `1`)
2. Скрипт подключится к аккаунту
3. Начнёт вступать в чаты из `config/accounts/1/chats.json`
4. **30 вступлений в день** (настраивается)
5. **Задержка 45-90 сек** между вступлениями
6. **Пауза 10-20 мин** после каждых 5 вступлений

## 📊 Пример расчёта

**Дано:**
- 3500 чатов
- 200 чатов/аккаунт
- 30 вступлений/день

**Расчёт:**
```
3500 чатов ÷ 200 = 17.5 → Нужно 18 аккаунтов

Для 1 аккаунта (200 чатов):
  200 чатов ÷ 30 вступлений/день = 6.7 дней
  
Итого: 7 дней на один аккаунт
```

**График для 1 аккаунта:**
```
День 1:  30 чатов  (остаток: 170)
День 2:  30 чатов  (остаток: 140)
День 3:  30 чатов  (остаток: 110)
День 4:  30 чатов  (остаток: 80)
День 5:  30 чатов  (остаток: 50)
День 6:  30 чатов  (остаток: 20)
День 7:  20 чатов  (готово!)
```

4. **Следите за ошибками** в progress.json

## 📈 Мониторинг прогресса

### **Файл progress.json:**

```json
{
  "joined": [
    {"id": "chat1", "name": "Чат 1", "joined_at": "2026-03-23T10:30:00"}
  ],
  "failed": [
    {"id": "chat2", "name": "Чат 2", "error": "ChannelPrivate"}
  ],
  "last_join_date": "2026-03-23",
  "joins_today": 15
}
```

**Поля:**
- `joined` — успешные вступления
- `failed` — ошибки (частные каналы, бан и т.д.)
- `last_join_date` — дата последнего вступления
- `joins_today` — сколько вступил сегодня

## ❌ Обработка ошибок

### **FloodWait**

```
⚠️ FloodWait 3600 сек для @chat_name
```

**Что делать:**
- Скрипт автоматически подождёт указанное время
- Если > 24 часов — чат помечается как "failed"

---

### **ChannelPrivate**

```
🔒 Частный канал: @chat_name
```

**Что делать:**
- Нужна ссылка-приглашение (t.me/+abc123)
- Чат помечается как "failed"

---

### **UserNotParticipantError**

```
🔒 Нужно приглашение: @chat_name
```

**Что делать:**
- Чат требует ручного подтверждения
- Помечается как "failed"

## 🎯 Полный цикл работы

```
1. python scripts/distribute_chats.py
   → Распределить 3500 чатов на 18 аккаунтов
   
2. python scripts/join_chats.py
   → Выбрать аккаунт 1
   → Вступить в 30 чатов (день 1)
   
3. На следующий день:
   → Запустить join_chats.py снова
   → Выбрать аккаунт 1
   → Вступить в 30 чатов (день 2)
   
4. Через 7 дней:
   → Аккаунт 1 готов (200 чатов)
   → Перейти к аккаунту 2
   
5. Через 126 дней:
   → Все 18 аккаунтов готовы
   → 3500 чатов доступны для рассылки
```
