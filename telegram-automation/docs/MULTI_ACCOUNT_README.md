# Multi-Account Telegram Broadcaster

Параллельная рассылка с нескольких аккаунтов Telegram с автоматическим распределением чатов.

## 🚀 Быстрый старт

### 1. Подготовка

```bash
# Убедитесь, что у вас есть .env с API credentials
# Если нет - скопируйте .env.example и заполните
cp .env.example .env
```

### 2. Добавление аккаунтов

**Вариант A: Через меню (рекомендуется)**

```bash
python multi_account_start.py
```

1. Выберите "Добавить аккаунт"
2. Выберите "Авторизовать аккаунт"
3. Введите телефон и код из Telegram

**Вариант B: Быстрая авторизация**

```bash
python auth_accounts.py
```

### 3. Настройка аккаунтов

Для каждого аккаунта настройте:

- **Скрипт** - текст сообщения (шаблон или свой)
- **Фото** - изображение для рассылки
- **Лимиты** - ограничения на рассылку

### 4. Запуск рассылки

```bash
python multi_account_start.py
```

Выберите "Запустить рассылку" → аккаунты автоматически распределят чаты!

## ⚙️ accounts.json

### Структура аккаунта

```json
{
  "id": 1,
  "name": "Аккаунт 1",
  "enabled": true,
  "session_name": "account_1",
  "api_id": 12345,
  "api_hash": "abc123def456",
  "phone": "+79990000001",
  
  "script": {
    "enabled": true,
    "template_id": 3,
    "custom_text": null
  },
  
  "photo": {
    "enabled": true,
    "use_default": true,
    "custom_path": null
  },
  
  "limits": {
    "use_global": true,
    "daily_limit": 500,
    "min_delay": 50,
    "max_delay": 100
  }
}
```

### Поля

| Поле | Описание |
|------|----------|
| `id` | Уникальный ID аккаунта |
| `name` | Отображаемое имя |
| `enabled` | Включён ли аккаунт |
| `session_name` | Имя файла сессии |
| `api_id`, `api_hash` | Telegram API credentials |
| `phone` | Телефон аккаунта |
| `script.enabled` | Использовать ли скрипт |
| `script.template_id` | ID шаблона из templates.json |
| `script.custom_text` | Свой текст (приоритетнее шаблона) |
| `photo.enabled` | Отправлять ли фото |
| `photo.use_default` | Использовать фото из templates.json |
| `photo.custom_path` | Путь к своему фото |
| `limits.use_global` | Использовать глобальные лимиты |
| `limits.daily_limit` | Дневной лимит сообщений |
| `limits.min_delay` | Мин. задержка между сообщениями (сек) |
| `limits.max_delay` | Макс. задержка между сообщениями (сек) |

## 🎯 Функции

### Автоматическое распределение чатов

Система автоматически распределяет чаты между аккаунтами:

- **Равномерно** - поровну между всеми аккаунтами
- **Пропорционально** - согласно лимитам каждого аккаунта

**Пример:**
- 1000 чатов, 2 аккаунта
- Аккаунт 1: 500 чатов
- Аккаунт 2: 500 чатов

### Индивидуальные настройки для каждого аккаунта

Каждый аккаунт может иметь:
- ✅ Свой текст сообщения (шаблон или кастомный)
- ✅ Своё фото
- ✅ Свои лимиты

### Масштабирование

- **Тестово**: 2 аккаунта
- **Рабочее**: 10-15 аккаунтов
- **Максимум**: зависит от ресурсов

## 🔧 Настройка скрипта для аккаунта

1. В главном меню выберите "Аккаунты"
2. Введите ID аккаунта для редактирования
3. Выберите "Настроить скрипт"
4. Опции:
   - Включить скрипт
   - Выбрать шаблон из templates.json
   - Ввести свой текст

## 🔧 Настройка фото для аккаунта

1. В главном меню выберите "Аккаунты"

3. Выберите "Настроить фото"
4. Опции:
   - Использовать фото по умолчанию (из templates.json)
   - Указать свой путь к фото
   - Отключить фото

## 🛡️ Безопасность

- Сессии хранятся локально в `sessions/`
- API credentials не передаются третьим лицам
- Рекомендуется использовать отдельные API ключи для каждого аккаунта

---

## ⚠️ Лимиты и ограничения

Telegram имеет ограничения на рассылку:

- **Дневной лимит**: ~500-1000 сообщений на аккаунт
- **Задержки**: 50-100 секунд между сообщениями
- **FloodWait**: при превышении лимитов

**Рекомендации:**
- Используйте 3-5 аккаунтов для 1000+ чатов
- Увеличивайте задержки при FloodWait
- Делайте перерывы между циклами рассылки

---

## 🐛 Решение проблем

### "Не авторизован"
→ Запустите авторизацию: `python auth_accounts.py`

### "FloodWait"
→ Увеличьте задержки в настройках аккаунта

### "Чаты пусты"
→ Проверьте `config/chats.json`

### "Аккаунт не настроен"
→ Заполните `api_id`, `api_hash`, `phone` в accounts.json

## 📊 Пример использования

### Сценарий: 1000 чатов, 2 аккаунта

1. **Добавляем аккаунты:**
   ```bash
   python multi_account_start.py
   → Добавить аккаунт (x2)
   → Авторизовать аккаунт (x2)
   ```

2. **Настраиваем скрипты:**
   - Аккаунт 1: Шаблон "greeting"
   - Аккаунт 2: Шаблон "offer"

3. **Настраиваем фото:**
   - Аккаунт 1: Фото по умолчанию
   - Аккаунт 2: Своё фото

4. **Запускаем рассылку:**
   ```bash
   python multi_account_start.py
   → Запустить рассылку
   → Равномерное распределение
   ```

5. **Результат:**
   - Аккаунт 1: 500 чатов отправлено
   - Аккаунт 2: 500 чатов отправлено
   - Итого: 1000 чатов за 1 цикл

## 📈 Расширение до 10-15 аккаунтов

1. Добавьте нужное количество аккаунтов через меню
2. Авторизуйте каждый аккаунт
3. Настройте индивидуальные скрипты и фото
4. Запустите рассылку - система автоматически распределит чаты

**Пример для 10 аккаунтов:**
- 5000 чатов ÷ 10 аккаунтов = 500 чатов на аккаунт
- Время рассылки: ~8-10 часов (с задержками 50-100 сек)
