# 🔌 Решение проблем с соединением Telegram

## ❗ Симптомы

```
Server closed the connection: 0 bytes read on a total of 8 expected bytes
RuntimeError: readexactly() called while another coroutine is already waiting for incoming data
```

## ✅ Что сделано в коде

В `start.py` добавлена обработка ошибок соединения:

1. **Автоматическое переподключение** при 3 ошибках подряд
2. **Экспоненциальная задержка** между попытками (2, 4, 8 сек)
3. **Продолжение рассылки** после успешного переподключения
4. **Логирование** всех ошибок соединения

---

## 🔧 Рекомендации для стабильной работы

### 1. Проверьте интернет-соединение

```bash
ping telegram.org
```

Если есть потери пакетов — проблема в интернете.

### 2. Используйте прокси (если Telegram заблокирован)

Добавьте в `config/config.json`:

```json
"proxy": {
  "enabled": true,
  "type": "http",
  "host": "proxy.example.com",
  "port": 8080,
  "username": "user",
  "password": "pass"
}
```

Или используйте SOCKS5:

```json
"proxy": {
  "enabled": true,
  "type": "socks5",
  "host": "127.0.0.1",
  "port": 9150
}
```

### 3. Увеличьте задержки между сообщениями

В `start.py` измените:

```python
MIN_DELAY = 120  # было 90
MAX_DELAY = 240  # было 180
```

### 4. Уменьшите лимиты сообщений

В `start.py` измените:

```python
DAILY_LIMIT = 30    # было 40
HOURLY_LIMIT = 8    # было 10
```

### 5. Проверьте сессию

Если сессия "протухла", пересоздайте её:

```bash
# Удалите старую сессию
del sessions\userbot.session

# Запустите авторизацию заново
python start.py → пункт 1
```

### 6. Обновите Telethon

```bash
pip install --upgrade telethon
```

### 7. Используйте стабильную версию Python

Проблемы могут быть с Python 3.13 (бета). Рекомендуется:
- Python 3.10 или 3.11

---

## 📊 Мониторинг ошибок

После рассылки проверьте лог:

```bash
logs/broadcast_log.json
```

Если видите много `connection_errors` — проблема в сети или блокировке.

---

## 🆘 Если ничего не помогает

1. **Подождите 1-2 часа** — временные проблемы на стороне Telegram
2. **Попробуйте другой аккаунт** — возможно, этот заблокирован
3. **Проверьте API ключи** — не совпадают с сессией
4. **Перезагрузите роутер** — смените IP адрес

---

## 📝 Логи для диагностики

Включите подробное логирование в `start.py`:

```python
import logging
logging.basicConfig(level=logging.DEBUG)
```

Или запустите с флагом отладки:

```bash
python -u start.py 2>&1 | tee debug.log
```

## ✅ Чек-лист стабильной рассылки

- [ ] Стабильный интернет (ping < 100ms)
- [ ] Задержки между сообщениями 90-180 сек
- [ ] Лимиты: 30-40 сообщений/день
- [ ] Свежая сессия (не старше 1 месяца)
- [ ] Telethon последней версии
- [ ] Python 3.10 или 3.11
