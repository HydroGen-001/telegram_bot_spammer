## Qwen Added Memories
- Принципы архитектуры Telegram-бота: 1) Модульность — разделение на независимые модули (core/client.py, core/broadcaster.py, core/multi_account.py), 2) start.py только для CLI меню, 3) ядро стабильно — логика отдельно от UI, 4) минимальные изменения в существующий код при расширении, 5) отдельный модуль для мультиаккаунтной рассылки без изменения основного файла, 6) приоритет SQLite для старта, PostgreSQL для масштабирования, 7) Docker для продакшена
- План разработки Telegram-бота: 1) Ротация IP между циклами (мобильный прокси, 30 мин), 2) JSON отчёты после каждого цикла, 3) Отправка статистики в ЛС, 4) Бот для получения номеров от людей. Приоритет: ротация → отчёты → ЛС → бот.
